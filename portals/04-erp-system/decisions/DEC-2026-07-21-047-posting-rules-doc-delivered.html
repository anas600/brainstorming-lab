🚨 اكتشافات حرجة (Critical Findings)

🔴 Finding #1: تناقض العملة (Currency Inconsistency)

المشكلة: EventPayload.Currency default = "SAR" (ريال سعودي). لكن السياق الليبي يستخدم LYD (دينار ليبي).

الموقع: IPostingRulesService.cs:22 → public string Currency { get; set; } = "SAR";

التأثير:
- لو Tenant سعودي → OK
- لو Tenant ليبي → Currency يُحفظ كـ SAR بدل LYD في كل event
- لكن العملة لا تُستخدم فعلياً في القيد (مخاطرة لاحقة)

التوصية: DEC منفصل لتحديد:
- هل النظام multi-currency؟
- هل الـ Currency من Tenant config؟
- أم default لكل region (SA/LY/EG)؟

🔴 Finding #2: لا يوجد Notification عند فشل القيد

المشكلة: PostingRulesService.ApplyRulesAsync:

if (totalDebit != totalCredit)
{
    _logger.LogError("قاعدة {RuleId} تنتج قيد غير متوازن: D={D} C={C} — تم التخطي",
        rule.Id, totalDebit, totalCredit);
    continue;  // ← يتخطى بصمت
}

التأثير:
- حدث يحدث في النظام
- النظام يحاول تطبيق قاعدة
- القيد غير متوازن → يتخطى بصمت
- المحاسب ما يعرف أبداً (لا notification، لا UI alert)
- الحدث يضيع

التوصية: إضافة notification عبر notifications table + إصلاح ليعطي 4xx response.

🔴 Finding #3: لا يوجد PUT Endpoint لتعديل القواعد

المشكلة: PostingRulesController عنده فقط GET + POST. لا يوجد PUT /api/finance/posting-rules/{id}.

التأثير:
- لو محاسب يبني قاعدة، ثم يحتاج يعدلها → لا يستطيع
- الحل الحالي: حذف وإنشاء جديد (لكن لا يوجد DELETE أيضاً!)
- الحل العملي: تعديل DB يدوياً

التوصية: إضافة PUT + DELETE endpoints (يستحق Sprint مستقل).

🟠 Finding #4: RBAC غير موثّق صراحةً

المشكلة: PostingRulesController:

[Authorize]  // ← فقط JWT، لا role-based

التأثير:
- أي مستخدم مسجّل يستطيع إنشاء/تشغيل القواعد
- لا يوجد تمييز بين Admin/Accountant/PM/Viewer

التوصية: DEC-048 منفصل لتوثيق RBAC الكامل.

🟠 Finding #5: EvaluateFormula محدود + Fallback صامت

المشكلة: فقط 4 صيغ مدعومة ({amount}, {amount}*2, رقم خام, فارغ). أي صيغة أخرى → 0 + log warning (صامت).

التأثير:
- المطور يكتب {amount}/2 → المبلغ = 0
- القيد قد يصبح غير متوازن → يتخطى (Finding #2)
- cascade failure

التوصية:
- توسيع EvaluateFormula (Phase 2 مخطط)
- إضافة validation قبل التطبيق (rejects invalid formulas)
