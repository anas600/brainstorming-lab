📁 موديل إدارة المشاريع (Projects Module) — شرح بالعربي

التاريخ: 2026-07-21
المؤلف: المختبر التحليلي (Mavis Lab)
الجمهور: الفريق غير التقني (أنس + صاحبه)
المصدر: live DB via Neon MCP

🎯 شو هو موديل المشاريع؟

موديل إدارة المشاريع مسؤول عن:

  • 🏗️ تعريف المشاريع (كود، اسم، ميزانية، تواريخ)
  • 📋 تقسيم المشاريع لمهام (tasks)
  • 👥 تعيين موارد (عمال، معدات) للمهام
  • 💰 تتبع الميزانية (مخطط vs فعلي)
  • 📊 حساب الربحية (لو المشروع لعميل)
  • 🔗 الربط مع Finance (المحاسبة) و HR (الموظفين)

بدون هذا الموديل:
- ما نعرف كم مشروع شغّال
- ما نعرف إذا الميزانية منفذة
- ما نعرف مين يشتغل على شنو
- ما نقدر نحسب تكلفة المشروع بدقة

🏗️ الطاولة المركزية: projects

شو هي؟

سجل المشاريع — كل مشروع = صف واحد فيه كل التفاصيل.

مثال حقيقي من DB:

الكود الاسم الميزانية الحالة الفترة
P-2026-001 مشروع بناء فيلات السُّرَّي 450,000.00 LYD 2 (جاري) 2026-01-10 → 2026-12-31
P-2026-002 تجديد مبنى municipal 180,000.00 LYD 2 (جاري) 2026-03-01 → 2026-10-31
P-2026-003 مشروع صيانة طرق 95,000.00 LYD 1 (مخطط) 2026-06-01 → 2026-12-31
P-2024-001 مشروع طريق المطار 50,000.00 LYD 0 (مكتمل) 2025-07-01 → 2025-11-01
P-2025-007 مجمع تجاري الشط 200,000.00 LYD 3 (مغلق) 2025-04-01 → 2025-09-01

الحقول المهمة:

الحقل المعنى مثال
code كود فريد للمشروع "P-2026-001"
name اسم المشروع (عربي) "مشروع بناء فيلات السُّرَّي"
description وصف تفصيلي (text طويل)
status الحالة (0-3) 2 (جاري)
budget الميزانية المرصودة 450,000.00
start_date تاريخ البداية 2026-01-10
end_date تاريخ النهاية المتوقع 2026-12-31
company_id الشركة المنفذة (multi-company) "uuid"
cost_center_id مركز التكلفة (للربط مع Finance) "uuid"
customer_id العميل (لو المشروع لعميل) "uuid" أو null
is_active هل نشط؟ true
deleted_at تاريخ الحذف الناعم (soft delete) null

الحالات (status):

status=0 → مكتمل (Completed)
status=1 → مخطط (Planned)
status=2 → جاري (In Progress)
status=3 → مغلق (Closed/Cancelled)

📋 المهام (project_tasks)

شو هي؟

كل مشروع ينقسم لـ مهام (tasks) — كل مهمة = خطوة واحدة في المشروع.

مثال:

المشروع: P-2026-001 (مشروع بناء فيلات السُّرَّي)

├── task 1: حفر الأساسات
│   ├── status: 2 (جاري)
│   ├── estimated_hours: 200
│   ├── actual_hours: 150
│   ├── start_date: 2026-01-15
│   └── end_date: 2026-02-15
│
├── task 2: بناء الهيكل الخرساني
│   ├── status: 1 (مخطط)
│   ├── estimated_hours: 400
│   ├── actual_hours: 0
│   ├── start_date: 2026-02-15
│   └── end_date: 2026-04-30
│
└── task 3: التشطيبات النهائية
    ├── status: 1 (مخطط)
    ├── estimated_hours: 300
    ├── actual_hours: 0
    ├── start_date: 2026-08-01
    └── end_date: 2026-12-31

الحقول:

الحقل المعنى
project_id ربط بالمشروع الأب
name اسم المهمة
description وصف المهمة
status 0=مكتمل، 1=مخطط، 2=جاري، 3=مغلق
estimated_hours الساعات المقدرة
actual_hours الساعات الفعلية
start_date تاريخ البداية
end_date تاريخ النهاية
progress_percent نسبة الإنجاز (0-100)

الفايدة: تقدر تشوف إذا المشروع في الوقت المحدد:
- actual_hours > estimated_hours → OVER BUDGET ⚠️
- progress_percent < expected → BEHIND SCHEDULE ⚠️

👥 الموارد والتعيينات (resources + resource_assignments)

شو هو المورد (resource)؟

كل "شي" ممكن يشتغل على مشروع:
- 👤 موظف (Human Resource)
- 🚜 معدة (Equipment) — حفار، رافعة
- 🔧 أداة (Tool) — خلاطة، مولد

مثال على resources:

الكود الاسم النوع السعر/ساعة نشط؟
R-001 حفار كاتربيلر 1 (معدة) 150.00 LYD ✅
R-002 عامل بناء خبرة 2 (إنسان) 25.00 LYD ✅
R-003 رافعة 30 طن 1 (معدة) 200.00 LYD ✅
R-004 مولد كهرباء 100kW 1 (معدة) 100.00 LYD ✅

resource_assignments (التعيينات):

ربط بين الموارد والمهام/المشاريع.

الحقل المعنى
project_id المشروع
task_id المهمة (لو محدد)
resource_id المورد (الحفار مثلاً)
user_id المستخدم (لو المورد إنسان)
from / to فترة التعيين
hourly_rate سعر الساعة (ممكن يختلف عن سعر المورد الأصلي)

الفايدة:
- حساب تكلفة المورد = hours × hourly_rate
- تتبع استخدام المعدات
- منع التعارضات (نفس المعدة في مشروعين بنفس الوقت)

💰 تتبع الميزانية (project_budgets)

شو هو؟

جدول منفصل للمتابعة الدقيقة للميزانية — يربط المشروع بحسابات محاسبية.

مثال حقيقي من DB:

المشروع: P-2026-001 (مشروع بناء فيلات السُّرَّي)
الميزانية: 450,000.00 LYD
المصرف: 0.00 LYD
المتعهد به: 0.00 LYD (committed)
المتبقي: 450,000.00 LYD

مرتبط بـ:
  - cost_center: "مشروع بناء فيلات السُّرَّي"
  - account: (حساب الميزانية)

الحقول:

الحقل المعنى
project_id المشروع
cost_center_id مركز التكلفة (في Finance)
account_id حساب الميزانية (في Finance)
budget_amount المبلغ المرصود
spent_amount اللي اتصرف فعلاً (من القيود)
committed_amount اللي اتعهد بصرفه (POs مفتوحة)
last_recalculated_at آخر تحديث

الفرق المهم:
- spent: اللي اتدفع فعلاً
- committed: اللي اتفق على دفعه (مثلاً PO مفتوحة)
- committed + spent ≤ budget → OK ✅
- committed + spent > budget → OVER BUDGET ⚠️

🔗 الربط بالموديلات الثانية

Projects ↔ Finance (الأقوى):

كل مشروع → cost_center في Finance
كل مصروف على المشروع → قيد يومي على cost_center
التقرير: "تكلفة المشروع" = مجموع القيود على cost_center

آلياً:
  - لما يصرف مصروف → يطلع قيد → cost_center = project.cost_center_id
  - spent_amount في project_budgets يتحدث

مثال عملي:

1. المشروع: P-2026-001 (مركز تكلفة T1)
2. شراء اسمنت بـ 5,000 LYD
3. النظام يعمل قيد:
   - مدين: مصروف مواد (5100) — T1
   - دائن: البنك (1202)
4. project_budgets.spent_amount يزيد بـ 5,000

التقرير: "مشروع P-2026-001 صرف 5,000 من 450,000"

Projects ↔ HR (الموظفين):

كل resource_assignments:
  - يربط resource بـ project
  - resource ممكن يكون user (من جدول users)
  - user عنده employee record (من HR module)

النتيجة: تقدر تشوف:
  - مين اشتغل على شنو
  - كم ساعة
  - التكلفة = hours × hourly_rate

Projects ↔ Procurement (المشتريات):

مشروع يحتاج مواد:
  1. تنشئ Purchase Order (PO)
  2. PO → items المطلوبة
  3. لما تستلم البضاعة → GRN
  4. GRN يحدث المخزون
  5. الفاتورة من المورد → قيد مالي
  6. القيد يذهب لـ cost_center الخاص بالمشروع

النتيجة: تكلفة المشروع = مشتريات + عمالة + مصاريف

📊 التقارير اللي تطلع من Projects

التقرير شو يقول
قائمة المشاريع كل المشاريع + حالتها + ميزانيتها
تفاصيل مشروع كل المهام، الموارد، المصروفات
تكلفة المشروع الفعلية مجموع المصروفات (من Finance cost_center)
ربحية المشروع الإيرادات - التكاليف (لو لعميل)
استخدام الموارد كم ساعة/يوم لكل مورد
المشاريع المتأخرة اللي progress_percent أقل من المتوقع

🎯 ملخص سريع

الجدول الدور
projects رأس المشروع (البيانات الأساسية)
project_tasks المهام الفرعية
project_budgets تتبع الميزانية (مخطط vs فعلي)
resources الموارد (عمال + معدات)
resource_assignments تعيين الموارد للمهام/المشاريع

الربط الأساسي: كل مشروع له cost_center_id — هذا هو الجسر لـ Finance.

الـ Workflow:
1. تنشئ مشروع → يولد cost_center تلقائياً
2. تضيف مهام → تقسيم العمل
3. تعيّن موارد → عمالة + معدات
4. تبدأ الشغل → مصاريف تطلع (Finance قيود → cost_center)
5. تخلص → تحسب الفعلي vs المخطط
